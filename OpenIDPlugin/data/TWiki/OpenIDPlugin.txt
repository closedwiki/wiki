---+ OpenID Plugin

This plugin implements both the Client and Server of OpenID. 
This will allow you to accept content from users that are registered on an OpenID implemented server, or to let other services authenicate using twiki registered users.

useful senario's include the slaving of registration data from the twiki.org server to the develop.twiki.org server, without requireing those user identities to be syncronised between the 2 servers

or to enable openid registered users to comment on your TWiki:Plugins.BlogUp based blog.

TWikiGroups are still defined in topics, but i'm not sure they will work ??

---++ Plugin Settings

Plugin settings are stored as preferences variables. To reference
a plugin setting write ==%<nop>&lt;plugin&gt;_&lt;setting&gt;%==, i.e. ==%<nop>OPENIDPLUGIN_SHORTDESCRIPTION%==

   * One line description, is shown in the %TWIKIWEB%.TextFormattingRules topic:
      * Set SHORTDESCRIPTION = Empty Plugin used as a template for new Plugins

   * Your own setting, for example:
      * Set EXAMPLE = got it!

   * Debug plugin: (See output in =data/debug.txt=)
      * Set DEBUG = 0

---++ Plugin Installation Instructions

   1 unarchive
   2 enable the functionality you need
      * select OpenIDUser.pm as your Password manager and OpenIDLogin as your LoginManager in configure to avoid the need for local registration of users (Consumer of Identifiers)
      * enable the OpenIDPlugin to run an OpenID server from your registered userbase (Identity Provider or Server)
      * *WARNING:* you cannot do both.
   3
