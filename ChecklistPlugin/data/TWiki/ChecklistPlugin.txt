---+!! <nop>%TOPIC%

%TOPIC% helps to create simple checklists. It handles the tags %<nop>CLI% (<nop>CheckList Item) and %<nop>CHECKLIST%.

A %<nop>CLI% tag creates a image button and a simple click on it will change the state of the image button. 
The default states are 'todo' (represented by a :-I image button) and 'done' (represented by a :ok: image button).


Every state change will be saved to a topic with name <i>&lt;YourTopicName&gt;</i><nop>ChecklistItemState. It is possible to 
define states (default: todo and done) and matching icons. You can also put more than one checklist into a topic.

---++ Usage Examples

Example 1: Create a simple checklist:
<verbatim>
	1 first step %CLI% 
	1 second step %CLI% 
	1 last step %CLI% 
</verbatim>

Example 2: Create a checklist with three states and own icons:
<verbatim>
%CHECKLIST{states="todo|doing|done" stateicons=":no:|:-I|8-)"%

	1 first step %CLI% 
	1 second step %CLI% 
</verbatim>

Example 3: Create two checklists on a topic (%X% every %<nop>CHECKLIST% and every %<nop>CLI% tag needs a name attribute):
<verbatim>
%CHECKLIST{name="first checklist"}%

  1 first step %CLI{name="first checklist"}% 
  1 second step %CLI{name="first checklist"}% 

%CHECKLIST{name="second checklist"}%

  1 first step %CLI{name="second checklist"}% 
  1 next step %CLI{name="second checklist"}% 
</verbatim>

Example 4: Create a checklist in a table:
<verbatim>
| *state* | *todo* |
| %CLI%	| first step |
| %CLI%	| second step |
</verbatim>

Example 5: Create a checklist without annoying line breaks:
<verbatim>
  1 %CLI{text="my first step"}%
  1 %CLI{text="my second step"}%
</verbatim>


---++ Syntax Rules

Just type a %<nop>CLI% (<nop>CheckList Item) into a topic to create a single checklist item.

To change the states or state icons you can set it for topic with a %<nop>CHECKLIST% tag.

Both tags can be used with attributes documented in the next section.

---++ Attributes
Usage: %<nop>CLI{attribute1="value1" attribute2="value2" ...}% <br/> or %<nop>CHECKLIST{attribute1="value1" attribute2="value2" ...}%

Examples:
<verbatim>
%CHECKLIST{showlegend="on" reset=":-I Reset checklist"}%
%CLI{id="i1"}%
</verbatim>

| *Attribute* | *Comment* | *Example* |
|  *%<nop>CLI{...}%*  |||
| id  | unique id of a checklist item (default: autogenerated); usefull and recommended if you insert new checklist items in an existing checklist with states | id="myfirstownid" |
| text | allows to place text behind the button (default: ""); usefull to avoid the annoying line break between checklist button and text | text="my first step" |
|  *%<nop>CHECKLIST{...}%*  |||
| reset | if defined it enables the reset image button with a given text or icon (default: undefined)| reset="Reset all" | 
| showlegend | if set to "on" a states legend with icons will be displayed (default: off),<br/> e.g: ( :-I - todo :ok: - done) | showlegend="off" |
|  *%<nop>CHECKLIST{...}% and %<nop>CLI{...}%*  |||
| name | a naming context (default: "_default") - useful and recommended if you define more than one checklist on a topic (%X% if you use it in %<nop>CHECKLIST% you must use it in all related %<nop>CLI% tags too) | name="mychecklist1" |
| states | all known states (default: "todo&brvbar;done"); a list separated by a '&brvbar;') | states="todo&brvbar;done" |
| stateicons | icon list with states separated by a '&brvbar;' (default: ":-I&brvbar;:ok:) | stateicons=":-I&brvbar;:ok:" |

---++ Plugin Settings

Plugin settings are stored as preferences variables. To reference a plugin setting write ==%<nop>&lt;plugin&gt;_&lt;setting&gt;%==, i.e. ==%<nop>INTERWIKIPLUGIN_SHORTDESCRIPTION%==

	* One line description, is shown in the %TWIKIWEB%.TextFormattingRules topic:
		* Set SHORTDESCRIPTION = helps to create simple checklists.

	* Debug plugin: (See output in =data/debug.txt=)
		* Set DEBUG = 0

	* Setup states and icons (defaults: <verbatim>todo|done</verbatim> and <verbatim>:-I|:ok:</verbatim>; remove # for setup):
		* # Set STATES = todo|done
		* # Set STATEICONS = :-I|:ok:	  

	* Enables 'reset' button (default: disabled with undefined value; value is a text or icon):
		* # Set RESET = [Reset all]

	* Enables state legend (default: disabled)
		* # Set SHOWLEGEND = off
