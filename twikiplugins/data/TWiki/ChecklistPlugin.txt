%TOPIC% helps to create simple checklists. It handles the tags %<nop>CLI% (<nop>CheckList Item) and %<nop>CHECKLIST%.

A %<nop>CLI% tag creates a image button and a simple click on it will change the state of the image button.

Every state change will saved to a topic with name <i>&lt;YourTopicName&gt;</i><nop>ChecklistItemState. It is possible to 
define states (default: todo and done) and matching icons. You can also put more than one checklist into a topic.


%TOC%

------
---++ Usage Examples

Example 1: Create a simple checklist:
<verbatim>
	1 %CLI% first step
	1 %CLI% second step
	1 %CLI% last step
</verbatim>

Example 2: Create a checklist with three states and own icons:
<verbatim>
%CHECKLIST{states="todo|doing|done" stateicons=":no:|:-I|8-)"%

	1 %CLI% first step
	1 %CLI% second step
</verbatim>

Example 3: Create two checklists on a topic:
<verbatim>
%CHECKLIST{name="first checklist"}%

  1 %CLI% first step
  1 %CLI% second step

%CHECKLIST{name="second checklist"}%

  1 %CLI% first step
  1 %CLI% next step

</verbatim>


---++ Syntax Rules

Just type a %<nop>CLI% (<nop>CheckList Item) into a topic to create a single checklist item.

To change the states or state icons you can set it for topic with a %<nop>CHECKLIST& tag before any %<nop>CLI% tag occurs.


---++ Attributes

| *Attribute* | *Comment* | *Example* |
| *%<nop>CLI{...}%* |||
| id  | unique id of a checklist item; usefull and recommended if you insert new checklist items in an existing checklist with states | id="myfirstownid" |
| *%<nop>CHECKLIST{...}%* |||
| name | a naming context - useful if you define more than one checklist on a topic | name="_default" |
| *%<nop>CHECKLIST{...}% and %<nop>CLI{...}%* |||
| states | all known states (separated by a '&brvbar;') | states="todo&brvbar;done" |
| stateicons | icon list with states (separated by a '&brvbar;') | stateicons=":no:&brvbar;:yes:" |
