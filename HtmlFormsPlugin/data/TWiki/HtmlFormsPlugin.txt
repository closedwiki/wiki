This plugin is currently useful to create a set of one change at a time web form elements - each form element has its own submit button, that goes to save - and it saves the new settings in META

---++ Syntax Rules

   * %!BUTTON{}%
   * %!CHECKBOX{}%
   * %!RADIO{}%
   * %!SELECT{}%
   * %!TEXTVALUE{}%
   * %!TEXTAREA{}%
   
---++++ common parameters
   * options="" - the comma seperated list of options, or VARIABLE containing or resulting in such a list (SEARCH is valid)
   * currentvalue="" - single value, or comma seperated list of values as appropriate
   * destinationaddress="Web.Topic.VARIABLENAME" (and where Web & Topic can be left out for the usual context driven locations

---++++ for later 
   * make all buttons submit all changes - just like inline edit!!
   * %!FORMVALUES{"Web.FormDefinitionTopic.FormfieldName"}% - returns a comma seperated list of options defined for that form field
   * later make a TOM like destination and source value syntax to get values from text, formfields etc
      * see http://twiki.org/cgi-bin/view/Codev/TopicObjectModel
